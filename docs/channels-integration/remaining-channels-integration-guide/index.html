Calendly OAuth Integration‚Äã

Overview‚Äã

Calendly uses OAuth 2.0 with backend-managed redirect flow. Frontend only needs to initiate auth and handle success/error redirects.

Flow‚Äã

1. Initiate Authentication

const handleCalendlyAuth = () => {
  window.location.href = `${API_BASE}/auth`;
};

2. Handle Backend Redirect

useEffect(() => {
  const params = new URLSearchParams(window.location.search);

  if (params.get("calendly_connected") === "true") {
    setStatus("‚úÖ Calendly connected successfully");
    loadConnectedAccounts();
    window.history.replaceState({}, "", window.location.pathname);
  }

  if (params.get("error")) {
    setError(decodeURIComponent(params.get("error")));
    window.history.replaceState({}, "", window.location.pathname);
  }
}, []);

Backend handles the OAuth callback, connects the account, and redirects to frontend with ?calendly_connected=true or ?error=....

Disconnect‚Äã

const handleDisconnect = async (accountId: number) => {
  await fetch(`${API_BASE}/disconnect`, {
    method: "POST",
    credentials: "include",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ accountId }),
  });
};

Endpoints‚Äã

EndpointMethodAuthPurpose
/channels/calendly/authGETSessionInitiate OAuth
/channels/calendly/callbackGETNoBackend callback (auto-redirects)
/channels/calendly/disconnectPOSTSessionRemove connection

2. Handle Redirect

useEffect(() => {

Telegram Integration‚Äã

Overview‚Äã

Telegram uses phone/2FA or QR authentication (not OAuth). The frontend drives multi-step flows by calling backend endpoints directly.

Phone Authentication Flow‚Äã

Step 1: Send Code‚Äã

const res = await fetch(`${API_BASE}/connect/phone`, {
  method: "POST",
  credentials: "include",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ phoneNumber }),
});

const { sessionId } = await res.json();

Step 2: Verify Code‚Äã

const res = await fetch(`${API_BASE}/connect/verify-code`, {
  method: "POST",
  credentials: "include",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ sessionId, code }),
});

const data = await res.json();

if (data.passwordNeeded) {
  // Proceed to 2FA
}

Step 3: Verify 2FA (if required)‚Äã

await fetch(`${API_BASE}/connect/verify-2fa`, {
  method: "POST",
  credentials: "include",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ sessionId, password }),
});

QR Authentication Flow‚Äã

Generate QR Code‚Äã

const res = await fetch(`${API_BASE}/connect/qr`, {
  method: "POST",
  credentials: "include",
});

const { qrUrl, sessionId } = await res.json();
  • Backend generates a Telegram login token.
  • Token is embedded in a tg://login?token=... QR code.
  • Token expires automatically (short TTL).

Poll QR Status (state-only)‚Äã

const interval = setInterval(async () => {
  const res = await fetch(
    `${API_BASE}/connect/qr/check?sessionId=${sessionId}`,
    { credentials: "include" }
  );

  const { status } = await res.json();

  if (status === "SUCCESS") {
    clearInterval(interval);
    loadConnectedAccounts();
  }

  if (status === "EXPIRED") {
    clearInterval(interval);
    // Regenerate QR
  }
}, 2000);

Status values

  • PENDING ‚Äì waiting for user to scan and accept
  • SUCCESS ‚Äì Telegram account connected
  • EXPIRED ‚Äì QR token expired, generate a new one

Important

  • This endpoint is read-only
  • It does not call Telegram APIs
  • Frontend never controls expiry timing

Endpoints‚Äã

EndpointMethodPurpose
/channels/telegram/connect/phonePOSTSend login code
/channels/telegram/connect/verify-codePOSTVerify login code
/channels/telegram/connect/verify-2faPOSTVerify 2FA password
/channels/telegram/connect/qrPOSTGenerate QR login session
/channels/telegram/connect/qr/checkGETPoll QR session status
/channels/telegram/disconnectPOSTDisconnect Telegram account

Key Differences‚Äã

OAuth Channels‚Äã

(Calendly, PayPal, Shopify, Stripe, Zoom, Facebook)

  • Redirect-based authentication

  • Backend handles OAuth callbacks

  • Success via query params:

    • ?{channel}_connected=true
    • ?error=...

Telegram‚Äã

  • No OAuth
  • Multi-step authentication
  • Frontend directly calls backend endpoints
  • QR login uses MTProto updates, not HTTP callbacks
  • Frontend polling is state-only
